## [v0.3.0] - 2026-05-24 ([`6fc7007`](https://github.com/ProductBuildersHQ/visionspec/commit/6fc7007))

### Highlights

- CLI composability - use visionspec as a library to build custom organization CLIs
- Configuration profiles for different product lifecycle stages (0-1, startup, growth, enterprise)
- Custom templates and rubrics with Loader interfaces and YAML support

### Added

- CLI as library (`pkg/cli`) with `AddCommandsTo()` and `Commands()` APIs
- Template Loader interface with EmbeddedLoader, FileLoader, EmbedFSLoader, ChainLoader
- Rubric Loader interface with YAML format support for custom rubrics
- SpecConfig types for configurable spec requirements per organization
- Profile system (`pkg/profiles`) with inheritance via `extends` field
- Default profiles: 0-1 (hypothesis), startup (PRD), growth (PRD+UXD+FAQ), enterprise (full suite)
- Profile CLI commands: `profiles list`, `profiles show <name>`
- `--profile` flag on `init` command for profile-based project initialization
- Example organization CLIs in `examples/` directory

### Changed

- Moved CLI commands from `internal/cli` to `pkg/cli` for library use
- Linter now accepts SpecConfig for organization-specific spec requirements
- Status command uses SpecConfig for determining required specs

### Documentation

- Phase 9 Composability added to ROADMAP.md
- Phase 10 Platform Enhancements roadmap with future features
- Example CLIs demonstrating profile-based organization tools
