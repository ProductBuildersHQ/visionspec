# v0.13.0 Release Notes

**Release Date:** 2026-07-18

This release introduces the **AIDLC Workflow Integration** and **Dual-Methodology System**, bringing full AWS AI-DLC workflow support to VisionSpec along with V2MOM strategic planning profiles and a new methodology architecture that separates requirements from implementation concerns.

## Highlights

- Full AIDLC workflow package with 12 document types across 3 phases
- Dual-methodology system: Requirements (WHAT to build) vs Implementation (HOW to build)
- V2MOM strategic planning profiles for company, department, and team levels
- AIDLC evaluation rubrics for LLM-as-judge document quality assessment
- V2 evaluation schema with 1-5 numeric scoring (replacing categorical pass/partial/fail)
- PIDL protocol export for workflow visualization

## What's New

### AIDLC Workflow Package (`pkg/aidlc/`)

Complete AWS AI-DLC workflow integration with support for all 12 document types:

**Inception Phase:**

- Vision Document
- Requirements Spec
- Technical Spec
- Architecture Spec

**Construction Phase:**

- Implementation Plan
- Test Plan
- Integration Plan
- Security Review

**Operations Phase:**

- Runbook
- Monitoring Plan
- Disaster Recovery
- SLO Document

```go
import "github.com/ProductBuildersHQ/visionspec/pkg/aidlc"

// Create default AIDLC workflow
workflow := aidlc.NewDefaultWorkflow()

// Check phase requirements
reqs := aidlc.GetPhaseRequirements(aidlc.PhaseInception)

// Attempt phase transition
result, err := workflow.TransitionTo(aidlc.PhaseConstruction)
```

### Dual-Methodology System

Separates methodology selection into two dimensions:

| Dimension | Purpose | Examples |
|-----------|---------|----------|
| Requirements Methodology | WHAT to build | AWS Working Backwards, Big Tech, Lean Startup |
| Implementation Methodology | HOW to build | AIDLC, SpecKit, GSD |

```go
type Project struct {
    RequirementsMethodology   string                    // "aws-working-backwards/product"
    ImplementationMethodology ImplementationMethodology // "aidlc", "speckit", "none"
}
```

### V2MOM Strategic Planning

New V2MOM spec types for organizational strategic alignment:

- `v2mom-vision` - Vision statement
- `v2mom-values` - Ranked values
- `v2mom-methods` - Strategic initiatives
- `v2mom-obstacles` - Blockers and risks
- `v2mom-measures` - Success metrics
- `v2mom-alignment` - Cascade alignment document
- `v2mom-summary` - Unified V2MOM view

V2MOM profiles for different organizational levels:

- `v2mom` - Generic V2MOM profile
- `v2mom-company` - Company-level strategic planning
- `v2mom-department` - Department-level V2MOM
- `v2mom-team` - Team-level V2MOM

### V2 Evaluation Schema

Evaluation now uses a 1-5 numeric scoring system:

| Score | Rating | Description |
|-------|--------|-------------|
| 5 | Excellent | Exceeds expectations |
| 4 | Good | Meets all requirements |
| 3 | Acceptable | Meets minimum requirements |
| 2 | Needs Work | Has significant gaps |
| 1 | Poor | Does not meet requirements |

```bash
# Evaluate with v2 schema
visionspec eval prd --format json
```

### AIDLC Evaluation Rubrics

12 new rubrics for evaluating AIDLC documents:

- `aidlc_vision.go` - Clarity, completeness, alignment, feasibility
- `aidlc_requirements.go` - Testability, traceability, ambiguity
- `aidlc_technical.go` - Architecture, scalability, security
- `aidlc_architecture.go` - ADR quality, component clarity
- `aidlc_implementation.go` - Milestones, tasks, resources, risks
- `aidlc_testplan.go` - Coverage, edge cases, automation
- `aidlc_integration.go` - Systems, contracts, migration
- `aidlc_security.go` - Threat model, controls, compliance
- `aidlc_runbook.go` - Step clarity, rollback, monitoring
- `aidlc_monitoring.go` - Metrics, logs, alerts, SLOs
- `aidlc_disaster.go` - RTO/RPO, procedures, testing
- `aidlc_slo.go` - SLI definitions, targets, error budgets

### PIDL Protocol Export

Export workflows to PIDL format for visualization:

```go
import "github.com/ProductBuildersHQ/visionspec/pkg/aidlc"

workflow := aidlc.NewDefaultWorkflow()
protocol := aidlc.WorkflowToPIDL(workflow)
```
